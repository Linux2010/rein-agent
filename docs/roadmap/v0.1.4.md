## Part 1: 网络访问工具 (P0)

### 1.1 WebFetch 工具

**功能**: 获取 URL 内容并用 prompt 处理

**参考**: `/Users/hope/ai-project/openclaude/src/tools/WebFetchTool/`

**实现方案**:

```typescript
// src/tools/web.ts
buildTool({
  name: 'web_fetch',
  description: 'Fetch content from a URL and process with a prompt',
  parameters: {
    type: 'object',
    properties: {
      url: { type: 'string', description: 'URL to fetch' },
      prompt: { type: 'string', description: 'Prompt to process content' },
    },
    required: ['url', 'prompt'],
  },
  execute: async (args) => {
    const response = await fetch(args.url);
    const content = await response.text();
    
    // Convert HTML to markdown (basic)
    const markdown = htmlToMarkdown(content);
    
    // Apply prompt (use LLM or simple extraction)
    const result = await applyPrompt(markdown, args.prompt);
    
    return { success: true, output: result };
  },
  isReadOnly: () => true,
});
```

**关键设计**:
- 预审批域名列表 (github.com, docs.google.com 等)
- 内容大小限制 (MAX_MARKDOWN_LENGTH)
- 错误处理 (404, timeout)

## Part 2: MCP 协议支持 (P0)

### 2.1 MCP 概述

**MCP (Model Context Protocol)** 是 Anthropic 定义的标准化工具协议：
- 工具定义 JSON Schema
- 工具调用标准格式
- 资源访问接口

**参考**: `/Users/hope/ai-project/openclaude/src/tools/MCPTool/`

### 2.2 MCP Tool 实现

```typescript
// src/tools/mcp.ts
interface MCPToolDefinition {
  name: string;
  description: string;
  inputSchema: object;  // JSON Schema
}

interface MCPClient {
  listTools(): Promise<MCPToolDefinition[]>;
  callTool(name: string, args: object): Promise<string>;
}

buildTool({
  name: 'mcp_call',
  description: 'Call an MCP tool from connected MCP servers',
  parameters: {
    type: 'object',
    properties: {
      server: { type: 'string', description: 'MCP server name' },
      tool: { type: 'string', description: 'Tool name to call' },
      args: { type: 'object', description: 'Tool arguments' },
    },
    required: ['server', 'tool', 'args'],
  },
  execute: async (args) => {
    const client = getMCPClient(args.server);
    const result = await client.callTool(args.tool, args.args);
    return { success: true, output: result };
  },
});

buildTool({
  name: 'mcp_list',
  description: 'List available MCP tools from connected servers',
  parameters: { type: 'object', properties: {} },
  execute: async () => {
    const tools = await listAllMCPTools();
    return { success: true, output: formatToolList(tools) };
  },
});
```

## Part 5: Bash 安全增强 (P1)

### 5.1 安全验证

**参考**: `/Users/hope/ai-project/openclaude/src/tools/BashTool/bashSecurity.ts`

**增强点**:
- 危险命令检测 (rm -rf, mkfs, dd)
- 沙箱模式选项
- 路径验证 (防止路径遍历)
- 只读命令预审批

```typescript
// src/tools/bash_security.ts
const DANGEROUS_PATTERNS = [
  /rm\s+-rf\s+\/$/,          // 删除根目录
  /mkfs/,                    // 格式化
  /dd\s+of=\/dev/,           // 磁盘写入
  /:\(\)\s*\{/,              // fork bomb
];

const READ_ONLY_COMMANDS = [
  'ls', 'cat', 'head', 'tail', 'grep', 'find', 'git status',
  'git log', 'git diff', 'npm list', 'node --version',
];

function isReadOnlyCommand(cmd: string): boolean {
  const baseCmd = cmd.split(' ')[0];
  return READ_ONLY_COMMANDS.includes(baseCmd);
}

function checkDangerousCommand(cmd: string): { safe: boolean; reason?: string } {
  for (const pat of DANGEROUS_PATTERNS) {
    if (pat.test(cmd)) {
      return { safe: false, reason: `Blocked dangerous pattern: ${pat}` };
    }
  }
  return { safe: true };
}
```
