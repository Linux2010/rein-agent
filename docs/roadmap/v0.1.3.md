# OpenHorse v0.1.3 Roadmap - Session 存储修复 & Memory 项目维度

## 背景

v0.1.3 包含两个核心改进：

1. **Session 存储修复** — tool_calls 数据缺失导致对话恢复断裂
2. **Memory 改为项目维度** — 与 OpenClaude 保持一致，每个项目独立记忆

## Part 1: Session 存储修复

### 问题分析

**当前存储格式（缺失关键信息）**:
```json
{"role":"assistant","content":"","timestamp":1778759247935,"toolCallId":"call_xxx"}
{"role":"tool","content":"{\"success\":true,\"output\":\"...\"}","timestamp":1778759247943,"toolCallId":"call_xxx"}
```

**缺失**:
- `tool_calls[].function.name` — 工具名
- `tool_calls[].function.arguments` — 调用参数

**LLM API 需要的格式**:
```json
{
  "role": "assistant",
  "content": "",
  "tool_calls": [
    {
      "id": "call_xxx",
      "type": "function",
      "function": {
        "name": "read_file",
        "arguments": "{\"path\":\"src/cli.ts\"}"
      }
    }
  ]
}
{
  "role": "tool",
  "content": "{\"success\":true,\"output\":\"...\"}",
  "tool_call_id": "call_xxx"
}
```

### 修复方案

#### 1.1 扩展 SessionMessage 类型

**文件**: `src/services/session-storage.ts`

```typescript
export interface ToolCallRecord {
  id: string;
  type: 'function';
  function: { name: string; arguments: string };
}

export interface SessionMessage {
  role: 'user' | 'assistant' | 'system' | 'tool';
  content: string;
  timestamp: number;
  toolCallId?: string;
  tool_calls?: ToolCallRecord[];  // 新增
}
```

#### 1.2 修改 handleChat 记录逻辑

**文件**: `src/commands/index.ts`

```typescript
let pendingToolCalls: ToolCallRecord[] = [];

case 'tool_call':
  pendingToolCalls.push({
    id: event.callId,
    type: 'function',
    function: {
      name: event.name,

case 'message':

```

#### 1.3 修改 loadSessionHistory

```typescript
export function loadSessionHistory(sessionId: string): Message[] {
  const messages = readSessionMessages(sessionId);
  return messages.map(m => ({
    role: m.role,
    content: m.content,
    ...(m.toolCallId ? { tool_call_id: m.toolCallId } : {}),
    ...(m.tool_calls ? { tool_calls: m.tool_calls } : {}),
  }));
}
```

## Part 2: Memory 项目维度重构

### 当前问题

OpenHorse 当前 memory 存储在 `~/.openhorse/memory/`（全局），与 OpenClaude 不一致。

**OpenClaude 设计**:
```
/Users/hope/.claude/projects/<project-path-hash>/memory/
  ├── MEMORY.md
  ├── user_role.md
  ├── feedback_style.md
  └── project_goals.md
```

### 新设计

**路径结构**:
```
~/.openhorse/projects/<project-path-hash>/memory/
  ├── MEMORY.md              # 索引文件
  ├── *.md                   # 记忆文件
```

**路径计算**:
```typescript
// 项目路径转为 hash（避免路径中的特殊字符）
import { createHash } from 'crypto';

function getProjectHash(projectPath: string): string {
  return createHash('sha256').update(projectPath).digest('hex').slice(0, 16);
}

// 例如: /Users/hope/ai-project/openhorse → a1b2c3d4e5f6g7h8
```

### 修改方案

#### 2.1 修改 memory/storage.ts

```typescript
import { createHash } from 'crypto';

/** 获取项目 hash */
export function getProjectHash(projectPath: string): string {
  return createHash('sha256').update(projectPath).digest('hex').slice(0, 16);
}

/** 获取项目的 memory 目录 */
export function getMemoryDir(projectPath?: string): string {
  const configHome = getConfigHome();  // ~/.openhorse
  
  if (projectPath) {
    const hash = getProjectHash(projectPath);
    return join(configHome, 'projects', hash, 'memory');
  }
  
  // 兼容：无 projectPath 时使用全局（v0.1.2 风格）
  return join(configHome, 'memory');
}

/** 确保项目 memory 目录存在 */
export function ensureProjectMemoryDir(projectPath: string): string {
  const dir = getMemoryDir(projectPath);
  if (!existsSync(dir)) {
    mkdirSync(dir, { recursive: true });
  }
  return dir;
}
```

```typescript
}
```
