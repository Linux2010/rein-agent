# OpenHorse v0.1.1 测试问题

## 已修复 ✅

### 问题1：输入交互后疯狂刷新 banner

**原因**：Spinner 每 80ms 调用 redraw() 清屏重绘整个界面

**修复方案**：使用 readline 标准方式
- 不清屏，直接追加输出
- Spinner 只更新当前行 `\r`
- 简单可靠，类似 OpenClaude 的标准 REPL

### 问题2：输出完全错乱，Spinner 和流式输出混在一起

**原因**：
1. cli.ts 和 handleChat 双重 spinner
2. spinner.stop() 清除行后，流式输出立刻写入同一行，混合显示

**修复方案**：
- cli.ts 不显示 spinner，让 handleChat 完全控制
- handleChat 在流式输出开始时，spinner.stop() 后打印换行 `\n`
- 让流式输出在新行开始，避免混合

**修改文件**: `src/cli.ts`, `src/commands/index.ts`

## 待修复

### 问题3：Turn X... 残留在模型输出中

**现象**：`⠙ Turn 2... (11.2s)  ▸ write_file  ✗ 1ms` spinner 文字和工具输出混合

**原因**：
- spinner 在工具执行期间继续运行
- spinner.stop() 只清除当前行，工具输出立刻写入同一行

### 问题4：缺少 think/action 信息展示

**现象**：长时间执行只显示工具名 `list_files`，没有 spinner 状态

**修复方案**：
- `tool_call` 事件时停止 spinner，换行
- `tool_result` 事件时显示工具结果，再启动 spinner
- spinner.stop() 使用 `\r\x1b[2K` 完全清除整行
- 工具输出前换行，避免和 spinner 同行

**修改文件**: `src/commands/index.ts`, `src/ui/box.ts`
