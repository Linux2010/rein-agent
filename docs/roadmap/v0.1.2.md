**核心策略**:
```typescript
const DEFAULT_MAX_RETRIES = 10
const MAX_529_RETRIES = 3
const BASE_DELAY_MS = 500

// Fast Mode Fallback
if (wasFastModeActive && (error.status === 429 || is529Error(error))) {
  // Short retry: wait and retry with fast mode
  // Long retry: enter cooldown, switch to standard model
  triggerFastModeCooldown(Date.now() + cooldownMs, cooldownReason)
  retryContext.fastMode = false
}

// Fallback Model
if (consecutive529Errors >= MAX_529_RETRIES && options.fallbackModel) {
  throw new FallbackTriggeredError(options.model, options.fallbackModel)
}
```

**重试分类**:
| 错误类型 | 策略 |
|---------|------|
| 429 Rate Limit | 等待 retry-after，继续 fast mode 或 fallback |
| 529 Overloaded | 短延迟重试，达到阈值后 fallback model |
| 401 Auth Error | 刷新 token，重建 client |
| ECONNRESET/EPIPE | 禁用 keep-alive，重新连接 |

Fork yourself when the intermediate tool output isn't worth keeping in context:

### Phase 1: Retry & Fallback System

#### 1.1 LLM Service 增强

**目标**: 完善错误处理与重试机制

**修改文件**: `src/services/llm.ts`

```typescript
interface RetryConfig {
  maxRetries: number;
  baseDelayMs: number;
  fallbackModel?: string;
  onRetry?: (attempt: number, error: Error) => void;
}

async function withRetry<T>(
  operation: () => Promise<T>,
  config: RetryConfig
): Promise<T> {
  let lastError: Error;
  for (let attempt = 1; attempt <= config.maxRetries; attempt++) {
    try {
      return await operation();
    } catch (error) {
      lastError = error;
      if (shouldRetry(error)) {
        const delay = exponentialBackoff(attempt, config.baseDelayMs);
        config.onRetry?.(attempt, error);
        await sleep(delay);
        continue;
      }
      throw error;
    }
  }
  throw lastError;
}
```

#### 1.2 Fallback Model 支持

**目标**: 主模型失败时自动切换备用模型

### Phase 3: Alternative Strategy

#### 3.1 Strategy Tracker

**目标**: 跟踪尝试过的策略，失败后尝试替代方案

interface StrategyAttempt {
  id: string;
  approach: string;
  toolsUsed: string[];
  result: 'success' | 'failed' | 'timeout';
  error?: string;
  duration: number;
}

interface StrategyTracker {
  attempts: StrategyAttempt[];

  maxAttempts: number;

  recordAttempt(attempt: StrategyAttempt): void;

#### 3.2 Query Loop 增强

**目标**: 在 query loop 中集成策略跟踪

**修改文件**: `src/framework/query.ts`

```typescript
export async function* query(params: QueryParams): AsyncGenerator<QueryEvent> {
  const tracker = new StrategyTracker({ maxAttempts: 3 });

  while (turn < maxTurns) {
    // 尝试当前策略
    const response = await llm.chatStream(messages, callbacks, tools);

    if (response.toolCalls) {
      for (const tc of response.toolCalls) {
        tracker.recordAttempt({
          approach: tc.function.name,
          toolsUsed: [tc.function.name],
          result: 'pending',
        });

        const result = await toolExecutor(tc.function.name, args);

        tracker.updateAttempt({
          result: result.success ? 'success' : 'failed',
          error: result.error,
        });

        // 如果失败，提示 LLM 尝试替代方案
        if (!result.success && tracker.isExhausted()) {
          messages.push({
            role: 'user',
            content: `Previous approach failed. Consider alternative strategies.`,
          });
        }
      }
    }
  }
}
```
